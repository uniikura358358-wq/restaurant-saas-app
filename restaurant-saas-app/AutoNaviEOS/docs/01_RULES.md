# 01_RULES.md（運用ルール：禁止事項・手順）

## 1. 作業ルール

- **事前許可**: コード、画像、動画の生成を行う際は、必ず事前にユーザーの許可を得ること。
- **文字コード**: 文字コードは `UTF-8` 固定とする。
- **変更記録**: 仕様変更が発生した場合は、必ず `04_DECISIONS.md` に理由と日付を記録し、関連する仕様書を更新すること。

## 2. ドキュメントルール

- **命名規則**: ファイル名は「番号＋英大文字（例：00_NAME.md）」とし、順序を固定する。
- **MVP表記**: 仕様書内では「最短導入セット」を「MVP」と表記するが、店舗向けドキュメントでは「MVP」という言葉を使わない工夫をすること。

## 3. 商品ルール

- **AutoNavi Wait (Pro)**: 通知はWebのみ（ブラウザプッシュ等）。
- **Premium**: 公式LINEがある場合のみ、LINEによる通知や連携機能を提供する。
- **SMS**: SMS送信は原価管理が必要なため、パック制などの別課金体系とする。

## 4. レビュー手順

開発は以下の順序でレビューを行いながら進行する：

1. 仕様 (Spec)
2. UI設計 (UI)
3. データ設計 (Data)
4. QA (Acceptance/TestCases)
5. 実装 (Implementation)
